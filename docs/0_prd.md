# 📍 [POI 随手记] 产品需求文档 (PRD)

## 1. 项目概述

本产品是一款个人地理信息管理工具，旨在帮助用户在移动端快速记录、管理和回顾具有特定意义的地点（POI）。用户可以通过地图直观地查看自己的足迹，并能通过分类、图片和文字详细记录每一个地点的独特属性。

---

## 2. 核心功能需求

### 2.1 地图交互与浏览

* **交互式地图底图**：App 主界面为全屏地图，支持手势缩放、平移和旋转。
* **地点标记显示**：所有已记录的地点以图标形式标在地图上。
* **密集点聚合**：当用户缩小地图导致地点图标重叠时，自动将邻近点聚合成一个带有数字的圆圈，点击圆圈可放大查看。


* **当前位置定位**：提供“回到中心”按钮，点击后地图自动跳转至用户当前所在的地理位置。
* **快速预览卡片**：点击地图上的地点图标，从屏幕底部弹出一个轻量化的预览卡片，显示地点名称、首张照片和分类，点击卡片进入详情。

### 2.2 地点（POI）录入与编辑

* **新增地点入口**：主界面提供显著的“新增”按钮。
* **位置捕获**：
* 默认自动获取当前设备所在的精确坐标。
* 支持用户在地图上通过拖拽或长按微调位置，以确保记录的准确性。


* **多维度属性填写**：
* **名称**：用户自定义地点的标题（必填）。
* **分类选择**：提供预设标签（如：工作、美食、打卡点、待办等），支持单选。
* **图库/相机接入**：用户可以直接现场拍摄照片，或从手机相册中选择已有照片关联至该地点。
* **文字备注**：支持输入详细的文字描述，记录当下的心情 or 备注事项。


* **数据保存**：保存后立即在地图上生成新的标记，并同步至云端存储。

### 2.3 地点详情展示

* **沉浸式详情页**：展示地点的全部信息。
* **图片轮播**：支持全屏查看图片，并可通过左右滑动切换多张照片。
* **属性列表**：清晰展示名称、分类、备注以及详细的经纬度信息。
* **记录时间**：自动显示该地点初次创建的时间。


* **外部导航联动**：详情页提供“导航”按钮，点击后唤起手机中已安装的其他地图软件（如自带地图、主流第三方导航软件），并自动将该点设为终点。

### 2.4 数据管理与搜索

* **列表视图**：除了地图模式，提供一个列表清单页面。
* **搜索功能**：支持通过关键字搜索地点名称或备注内容。
* **排序功能**：支持按“最近添加”或“距离当前位置最近”进行排序。


* **删除与修改**：用户可以随时修改已记录的点位信息，或删除不再需要的记录。

### 2.5 账号与同步体系

* **用户注册与登录**：支持邮箱/密码形式注册及登录。
* **数据云同步**：POI 数据（名称、坐标、备注、分类、图片 URL）实时同步至 Supabase 服务器，确保多端一致。
* **图床 API 配置**：用户在设置中输入自己的 ImgBB API Key。App 仅作为客户端调用，数据不经过开发者服务器。

### 2.6 离线地图管理

* **自动缓存**：用户在有网络环境下浏览地图时，App 自动将浏览过的瓦片数据缓存至本地 MBTiles 数据库中。
* **区域下载**：支持用户在地图上框选特定区域，一键下载该区域的 0~12 级瓦片数据，确保在无网环境下也能查看基础底图。
* **空间管理**：显示当前缓存占用的存储空间，支持一键清理缓存。

---

## 3. 业务规则与限制

### 3.1 存储与流量优化 (用户感知层面)

* **图片处理策略**：App 在上传至 ImgBB 前会自动进行智能压缩（WebP 格式），在保证视觉清晰度的前提下减小文件体积。
* **存储归属**：图片文件存储在用户个人的 ImgBB 空间中，App 仅记录返回的 URL。
* **云同步**：POI 文本与元数据实时备份，用户更换设备后登录账号并配置图床 Key 即可找回全部记录。

### 3.2 异常场景处理

* **无定位权限**：若用户拒绝定位授权，App 应提示用户无法自动获取位置，并引导其手动在地图上选点。
* **离线环境**：在没有网络的情况下，允许用户先录入文字信息和照片，待网络恢复后自动完成同步。

---

## 4. 用户交互设计要求

* **极简原则**：从打开 App 到完成一次地点记录，点击次数应控制在 4 次以内。
* **操作反馈**：所有保存、删除、上传操作均需有明确的轻提示（如：保存成功、上传中...）。
* **视觉风格**：地图界面应保持清爽，避免不必要的 UI 遮挡地图视野。

---

## 5. 后续规划 (Roadmap)

* **数据导出**：支持将所有记录导出为表格文件（Excel）或通用地理格式。
* **足迹分享**：支持生成一张精美的足迹海报分享至社交平台。
* **围栏提醒**：当用户经过曾经记录过的某个 POI 附近时，推送消息提醒。

## 6. 核心界面布局与功能详细说明

### 6.1 首页：沉浸式地图看板 (Home View)
这是应用的核心入口，采用“全屏底图+浮动控件”的结构。

#### 界面布局：

[顶层-悬浮栏]：搜索框横跨顶部（左右留边 16px），内置“搜索点位/备注”占位符，右侧带头像图标。

[中层-地图区]：展示 MapLibre 矢量底图，覆盖全屏。

[右侧-工具链]：在屏幕右侧中部偏下，垂直排列两个白色圆形按钮：

图标：图层：点击弹出底图切换（标准/卫星/地形）。

图标：指南针：点击重置地图朝向。

[右下角-核心区]：

定位按钮：白色，点击回到用户当前经纬度。

新增按钮：品牌蓝色，直径 64px，带有明显的投影。

#### 关键功能描述：

动态聚合 (Clustering)：当缩放级别较低时，重叠的 POI 点自动变为带数字的圆圈。

长按触发 (Quick Add)：长按地图任意位置 1 秒，系统给予震动反馈，并在该点自动弹出“在此处录入”的气泡。

### 6.2 POI 详情抽屉 (Bottom Sheet / Drawer)
点击 Marker 后从屏幕底部平滑升起的交互层。

#### 界面布局：

[顶部手柄]：居中的灰色短横线，提示用户可以上下滑动。

[头部区]：左侧显示名称（H1），右侧显示一键“导航”按钮。

[图片区]：横向滚动的相册，高度固定（如 200px）。

[属性区]：

第一行：分类标签（如：🚩 徒步路线）。

第二行：距离（如：距离当前位置 500m）。

[详情区]：展示多行文本备注。

[底部动作条]：水平排列“编辑”、“分享”和红色“删除”文字链接。

#### 关键功能描述：

三段式交互：

轻触 Marker：抽屉升至 30%（Peek），展示核心属性。

向上扫动：抽屉升至 90%（Full），展示完整图片和长文本。

点击地图空白：抽屉完全收回。

### 6.3 POI 极速录入页 (Add Form)
强调“先保存，后上传”的极简采集体验。

#### 界面布局：

[导航栏]：左侧“取消”，右侧“保存”按钮。

[图片采集区]：位于页面最上方，大型“相机”图标按钮，点击后占据半个屏幕预览实时画面或已选照片。

[核心字段区]：

输入框：地点名称（必填，自动获取焦点）。

分类选择：横向滚动的 Chip 标签（如：餐厅、摄影、待办）。

备注框：多行文本输入。

[位置微调区]：展示一个小型的正方形静态地图，显示当前捕获的点位，点击可全屏重选。

#### 关键功能描述：

后台异步上传：点击“保存”后，页面立即关闭并提示“已进入上传队列”。即使用户立即锁屏，Tauri Rust 后端也会继续完成图片压缩与图床上传。

自动分类记忆：系统自动记录用户最常用的 3 个分类，并排在标签栏最前端。

### 6.4 列表与管理视图 (List View)
为用户提供按数据维度检索的入口。

#### 界面布局：

[顶部切换器]：两段式开关，用于在“地图”与“列表”模式间快速切换。

[列表项]：

左侧：正方形缩略图。

右侧：上方名称，下方距离与时间戳。

状态角标：若处于“等待上传”状态，显示灰色时钟图标。

#### 关键功能描述：

多维排序：支持“按时间最新”和“按距离最近（动态实时计算）”。

侧滑快捷操作：在列表项上向左滑动，露出“删除”和“隐藏”按钮。

### 6.5 登入与注册页 (Auth View)

#### 界面布局：
* [Logo 区]：顶部展示品牌 Icon 与应用名称。
* [表单区]：包含邮箱输入框、密码输入框。
* [动作区]：主按钮“登录”、副按钮“注册新账号”。

### 6.6 设置中心 (Settings View)

#### 界面布局：
* **[个人资料]**：显示当前登录账号，支持登出。
* **[图床设置]**：
    * 标题：“ImgBB 开发者设置”。
    * 输入框：API Key（支持粘贴，默认掩码显示）。
    * 状态：显示 API Key 是否有效。
* **[离线包管理]**：
    * 按钮：“选择区域下载地图”。
    * 进度条：显示当前下载百分比。
    * 存储统计：已占用空间（如：124.5 MB）。
